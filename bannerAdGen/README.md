# Banner Ad Builder

This is a local, browser-based tool for editing the bookstore promotional banner JavaScript without manually changing banner objects in the code.

Open `index.html` in a browser. No backend, MBS access, server access, login, or build step is required.

## What the existing banner file does

The live script keeps banner ads in a `const promos = [...]` array. Each item is one banner. After that array, the script filters out expired or disabled banners, gives more weight to promotional and seasonal banners, randomly selects banners for the `promoTop` and `promoBottom` containers, renders responsive images, and sends GA4 click tracking.

## Banner data model

Each banner entry currently uses these fields:

- `promoName`: internal label and GA promotion name.
- `url`: click destination. The builder creates this from a destination URL plus UTM tracking.
- `alt`: accessible description for the banner image.
- `desktop`: desktop banner image path, currently intended for 1300x100 display.
- `mobile`: mobile banner image path, currently intended for 768x150 display.
- `use`: rotation category. Supported values are `evergreen`, `seasonal`, `promotional`, and `disabled`.
- `endDate`: last eligible date for the banner, written as `YYYY-MM-DD`.

Priority is controlled by list order and by the existing weighting logic. Active status is controlled by `use` and `endDate`: `disabled` banners do not show, and expired banners do not show.

The current file does not support separate headline, subheadline, CTA text, start date, background color, or theme fields. Those details are part of the banner image itself or the destination URL.

## How to use it

1. Open `index.html`.
2. Click **Load .js** and choose the current banner JavaScript file, or paste the file contents into the Current JavaScript box.
3. Click **Parse Banners**.
4. Select a banner from the list to edit it, or click **Add Banner**.
5. Enter a destination URL and campaign name. The builder automatically adds `utm_source=onsite`, `utm_medium=banner`, and `utm_campaign`.
6. Fill in the remaining required fields and click **Add Banner** or **Update Banner**.
7. Use the up/down buttons to reorder banners.
8. Use **Disable** to keep a banner in the file but remove it from rotation.
9. Use **Delete** only when you want the banner removed from the exported array.
10. Review the preview and Updated JavaScript output.
11. Click **Copy JS** or **Download JS**. The downloaded file is named `random-banner.js`.

## Exporting updated JavaScript

When a valid source file is loaded, the tool replaces only the `const promos = [...]` array and preserves the rest of the script as practical. That means the filtering, rendering, random selection, temporary boosts, and GA tracking code remain outside the form editor.

After exporting, replace the website's current banner JavaScript with the updated JavaScript content using your normal publishing process.

## Safety notes and limitations

- The form treats banners as structured data, not arbitrary JavaScript.
- CTA URLs are generated by the URL builder. Staff enter the destination URL and campaign name; the tool appends `utm_source=onsite`, `utm_medium=banner`, and `utm_campaign`.
- User-entered text is escaped with JSON string formatting so quotes and apostrophes do not break the exported file.
- The tool validates required fields before adding or updating a banner.
- The tool intentionally does not edit the random selection logic, GA tracking, date filtering, temporary weighting boosts, or HTML rendering function.
- Existing inline comments inside banner objects may be simplified into an optional comment above the exported banner entry.
- If a future banner file changes away from `const promos = [...]`, parsing may fail. In that case, use the exported banner array as a safer snippet and paste it into the file manually.
